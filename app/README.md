# SmartRec Backend API

Серверная часть системы интеллектуального управления образовательными программами ITMO SmartRec. API обеспечивает взаимодействие между реляционной базой данных (PostgreSQL), графовой базой (Neo4j) и фронтенд-приложением.

## 🛠 Стек технологий

*   **Фреймворк:** FastAPI (Python 3.10+)
*   **Сервер:** Uvicorn
*   **Базы данных:**
    *   **PostgreSQL:** Хранение строгих реляционных данных через драйвер `psycopg2`.
    *   **Neo4j:** Анализ связей и рекомендательный движок через официальный `neo4j` Python driver.
*   **Безопасность:** JWT-токены, OAuth2 (Password flow).

## 📂 Структура проекта (папка `app/`)

*   `main.py` — Точка входа в приложение, настройка CORS и подключение роутеров.
*   `database.py` — Управление соединениями с БД (пулы коннектов для Postgres и драйвер для Neo4j).
*   `auth.py` — Логика аутентификации, генерация и валидация JWT-токенов, управление правами доступа (Админ / Студент).
*   `routes/` — Эндпоинты API:
    *   `auth.py` — Роуты для входа и получения токена (`/api/auth/token`, `/api/auth/me`).
    *   `recommendations.py` — **Ядро системы**. Сложные запросы к Neo4j для генерации персональных образовательных траекторий.
    *   *(Остальные роуты управляют планами, треками и прогрессом студентов)*

## 🧠 Логика рекомендательного движка

Логика формирования рекомендаций сосредоточена в `routes/recommendations.py`.
1.  Определяется текущий семестр студента на основе года его поступления (связь `STUDIES_ON` -> `Program.year`).
2.  Cypher-запрос к Neo4j обходит граф: `Student -> Program -> Module -> Discipline`.
3.  Для каждой доступной дисциплины проверяются входные требования (связи `REQUIRES`). Система агрегирует статусы `COMPLETED` для всех пререквизитов.
4.  Дисциплины, для которых студент не закрыл пререквизиты, отфильтровываются.
5.  На уровне Python-кода (после возврата данных из БД) дисциплины группируются по типу выборности: обязательные идут отдельным списком, а элективы объединяются по названию модуля.

## 🚀 Локальный запуск (вне Docker)

1. Установите зависимости (предпочтительно в виртуальном окружении):
   ```bash
   pip install -r ../requirements.txt
   ```
2. Убедитесь, что PostgreSQL и Neo4j запущены (обычно через `docker-compose up -d db neo4j`).
3. Задайте переменные окружения (или они возьмутся по умолчанию для локальной разработки):
   * `DATABASE_URL=postgresql://postgres:rootpassword@localhost:5433/itmo_db`
   * `NEO4J_URI=bolt://localhost:7687`
4. Запустите сервер:
   ```bash
   uvicorn app.main:app --reload --port 8000
   ```
5. Документация Swagger доступна по адресу `http://localhost:8000/docs`.
