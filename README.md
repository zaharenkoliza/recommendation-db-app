# ITMO Educational Program Management System (Semantic Extension)

Информационная система управления образовательной программой с гибридной архитектурой (PostgreSQL + Neo4j). Система позволяет не только хранить данные об учебных планах, но и проводить семантический анализ программы, проверять пререквизиты и давать интеллектуальные рекомендации студентам.

## 🏗 Архитектура

Проект состоит из трех основных компонентов:
1.  **PostgreSQL (Relational Layer)**: Хранит основные структурированные данные: учебные планы (Curricula), дисциплины, данные студентов и преподавателей, а также историю успеваемости.
2.  **Neo4j (Knowledge Graph Layer)**: Используется как графовая надстройка. Позволяет строить сложные цепочки зависимостей (пререквизиты), анализировать траектории и формировать рекомендации.
3.  **FastAPI (Backend API)**: Сервер на Python, который объединяет обе базы данных и предоставляет эндпоинты для фронтенда.

## 🚀 Быстрый старт

### Предварительные требования
*   Docker и Docker Compose
*   Python 3.10+ (для запуска скриптов миграции вне контейнера, если потребуется)

### Запуск проекта
1.  Склонируйте репозиторий и перейдите в папку проекта:
    ```bash
    cd d:/ITMO/БД/project
    ```
2.  Запустите все сервисы через Docker Compose:
    ```bash
    docker-compose up -d --build
    ```
3.  После запуска базы данных будут доступны по адресам:
    *   **Backend API**: [http://localhost:8000/docs](http://localhost:8000/docs) (Swagger UI)
    *   **Neo4j Browser**: [http://localhost:7474](http://localhost:7474) (login: `neo4j`, pass: `password`)
    *   **PostgreSQL**: `localhost:5432` (db: `itmo_db`, user: `s335141`)

## 🔄 Синхронизация данных (Migration)

Для переноса данных из реляционной базы в граф используется скрипт `migrate.py`. Он автоматически создает узлы и связи (Program, Track, Module, Discipline, Student) и восстанавливает семантическую структуру.

Чтобы запустить синхронизацию вручную:
```bash
python migrate.py
```

## 🧠 Интеллектуальные функции

В системе реализован алгоритм подбора курсов (в `app/routes/recommendations.py`), который учитывает:
*   **Трек студента**: подбираются курсы, входящие в выбранную специализацию.
*   **Историю успеваемости**: исключаются уже пройденные дисциплины.
*   **Граф пререквизитов**: курс рекомендуется только если студент уже закрыл все необходимые входные требования (связи `REQUIRES` в Neo4j).

**Как формируется объяснение рекомендации (Explainability):**
Помимо самого списка курсов, бэкенд возвращает причину (поле `reason`), объясняющую, почему именно этот курс был предложен:
1.  **Cypher Запрос**: В базе Neo4j во время поиска доступных дисциплин происходит агрегация всех входных пререквизитов (`all_reqs`) и их фильтрация по статусу `COMPLETED` у студента.
2.  **Генерация ответа**: В Python скрипте проверяется список выполненных пререквизитов. Если он не пуст, формируется строка вида: `"Доступно на основе: Математика, Информатика"`. Если курс является базовым и не имеет зависимостей, возвращается: `"Базовая дисциплина (нет пререквизитов)"`.
3.  **Отображение**: Фронтенд принимает этот текст и выводит в карточке курса, повышая прозрачность работы рекомендательной системы для конечного пользователя.

## 📝 Схема данных (SQL)
Основные таблицы в схеме `s335141`:
*   `curricula` — учебные планы (программы).
*   `tracks` — специализации (треки).
*   `disciplines` — каталог дисциплин.
*   `student_progress` — успеваемость студентов.
*   `discipline_prerequisites` — связи между предметами.
