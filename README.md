# ITMO SmartRec (Educational Program Management System)

Информационная система управления образовательной программой с гибридной архитектурой (PostgreSQL + Neo4j). Система позволяет не только хранить данные об учебных планах, но и проводить семантический анализ программы, проверять пререквизиты и давать интеллектуальные рекомендации студентам.

## 🏗 Архитектура

Проект состоит из трех основных слоев:
1.  **PostgreSQL (Relational Layer)**: Хранит основные структурированные данные (Учебные планы, дисциплины, треки, оценки студентов). Гарантирует консистентность данных. Схема создается через `init.sql`.
2.  **Neo4j (Knowledge Graph Layer)**: Используется как графовая надстройка. Строит сложные цепочки зависимостей (пререквизиты), анализирует траектории и формирует рекомендации. Граф генерируется из SQL базы с помощью скрипта `migrate.py`.
3.  **FastAPI + React**: Бэкенд, объединяющий базы данных, и современный фронтенд для удобного взаимодействия студентов и администраторов.

## 🚀 Быстрый старт (Docker Compose)

Вся система завернута в Docker Compose для простого запуска одной командой.

### Запуск проекта
1.  Склонируйте репозиторий и перейдите в папку проекта.
2.  Запустите все сервисы через Docker Compose:
    ```bash
    docker-compose up -d --build
    ```
3.  После сборки и запуска контейнеров будут доступны:
    *   **Frontend (UI)**: [http://localhost:3000](http://localhost:3000)
    *   **Backend API**: [http://localhost:8000/docs](http://localhost:8000/docs) (Swagger UI)
    *   **Neo4j Browser**: [http://localhost:7474](http://localhost:7474) (login: `neo4j`, pass: `password`)
    *   **PostgreSQL**: `localhost:5433` (db: `itmo_db`, user: `postgres`, password: `rootpassword`)

## 🔄 Синхронизация данных (ETL)

Для переноса данных из реляционной базы в граф используется скрипт `migrate.py`. Запускать его нужно, если вы изменили данные в PostgreSQL (например, импортировали новые учебные планы) и хотите обновить граф рекомендаций.

```bash
# Для запуска внутри контейнера бэкенда (рекомендуется)
docker exec -it itmo_backend python migrate.py

# Для генерации "Умных пререквизитов" (с применением семантического анализа)
docker exec -it itmo_backend python build_smart_prerequisites.py
```
