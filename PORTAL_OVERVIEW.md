# ITMO Academic Portal — Обзор возможностей

## Архитектура системы

| Компонент | Технология | Назначение |
|-----------|-----------|-----------|
| Backend | FastAPI (Python) | REST API, бизнес-логика |
| Frontend | React + TypeScript (Vite) | Веб-интерфейс |
| БД реляционная | PostgreSQL | Хранение учебных данных, оценок, структуры учебных планов |
| БД графовая | Neo4j | Граф зависимостей дисциплин, рекомендации |
| Оркестрация | Docker Compose | Запуск всех сервисов |

**Сервисы:**
- Backend: `http://localhost:8000` (Swagger: `/docs`)
- PostgreSQL: `localhost:5433`
- Neo4j: `localhost:7474` (HTTP) / `localhost:7687` (Bolt)
- Frontend: `http://localhost:3000`

---

## Аутентификация

- Тип: **JWT (Bearer Token)**, срок жизни 24 часа
- Эндпоинт: `POST /auth/login` — логин/пароль → токен
- Получение текущего пользователя: `GET /auth/me`
- Роли: `ADMIN`, `STUDENT`
- Токен хранится в `localStorage` браузера
- При 401-ошибке — автоматический выход из системы

**Тестовый пользователь (showcase):**
- Логин: `maksim` / Пароль: `password` — роль `STUDENT`

## Аккаунт студента

### 1. Главная страница — Рекомендации (`/`)

**Что отображается:**
- Шапка с данными студента: ФИО, ID, группа, трек, учебный план, курс
- Карточки со статистикой: специализация, учебный план, номер курса, количество рекомендаций, кредиты
- Сетка карточек рекомендованных дисциплин

**Карточка рекомендации содержит:**
- Название и ID дисциплины
- Количество пройденных пререквизитов
- Причину рекомендации:
  - `СРОЧНО: Академическая задолженность` — дисциплина провалена, нужна пересдача
  - `Доступно на основе: [список пройденных пред.]` — все пререквизиты выполнены
  - `Базовая дисциплина (нет пререквизитов)` — пререквизитов нет
- Красный индикатор, если дисциплина — академическая задолженность
- Кнопка перехода к графу зависимостей дисциплины

**Как работает алгоритм рекомендаций:**
1. Из Neo4j получается граф: `Студент → Программа → Модули → Дисциплины`
2. Исключаются уже пройденные дисциплины
3. Проверяется выполнение пререквизитов (`REQUIRES`)
4. Приоритет: сначала задолженности (`HAS_DEBT`), затем доступные дисциплины
5. Возвращается до 10 рекомендаций

**API:** `GET /api/recommend/{student_id}`

---

### 2. Мой учебный план (`/my-curriculum`)

**Что отображается:**
- Заголовок плана (название, степень, год, руководитель)
- Иерархическое дерево разделов: **Учебный план → Разделы → Модули → Дисциплины**
- Каждый раздел/модуль раскрывается/сворачивается
- Для модулей: ID, позиция, тип выбора, лимит выбора
- Для дисциплин: название, номер семестра, ссылка на граф зависимостей
- Выпадающий список быстрого перехода к дисциплине

**API:** `GET /student/curriculum`

---

### 3. Прогресс (`/progress`)

**Что отображается:**
- Сводные карточки: всего дисциплин, сдано, задолженности, процент сдачи
- Анимированный прогресс-бар
- Три раскрываемые секции:
  - **Задолженности / Провалено** — раскрыта по умолчанию при наличии
  - **Успешно сдано** — раскрыта по умолчанию если нет задолженностей
  - **Зачислен на курс** — свёрнута по умолчанию
- Каждая запись: название дисциплины, номер попытки, оценка (2–5 с текстом), ссылка на граф

**Цветовая схема оценок:**
- Красный — 2 (Неудовлетворительно)
- Жёлтый — 3 (Удовлетворительно)
- Зелёный — 4, 5 (Хорошо / Отлично)

**API:** `GET /students/{student_id}/progress`

---

### 4. Граф зависимостей дисциплины

> Доступен и студентам, и администраторам (переход по ссылке из любой страницы с дисциплинами)

**Что отображается:**
- Интерактивный граф (ReactFlow):
  - Вверху — пререквизиты (что нужно сдать ДО)
  - В центре — выбранная дисциплина (синяя)
  - Внизу — зависимые дисциплины (что откроется ПОСЛЕ)
- Цветовая маркировка узлов: синий = цель, светло-синий = пререквизиты, серый = зависимые
- Анимированные рёбра-связи

**API:** `GET /admin/disciplines/{disc_id}/graph`

## Аккаунт администратора

### Навигация
Администратору дополнительно доступно меню `/admin/*`. Роль отображается как фиолетовый бейдж `Admin` в сайдбаре.

Также администратор может переключаться между студентами для просмотра их рекомендаций и прогресса (компонент `StudentSelector`).

---

### 1. Управление учебными планами (`/admin/curricula`)

**Возможности:**
- Просмотр списка всех учебных планов
- Поиск/фильтрация по названию и году
- Статистика: всего, бакалавриат, магистратура
- Таблица: ID ИСУ, Название, Год, Степень, Руководитель

**CRUD операции:**

| Действие | HTTP | Эндпоинт | Поля |
|----------|------|----------|------|
| Создать | `POST` | `/admin/curricula` | id_isu, name, year, degree, head |
| Обновить | `PUT` | `/admin/curricula/{id_isu}` | name, year, degree, head |
| Удалить | `DELETE` | `/admin/curricula/{id_isu}` | — (каскадное удаление) |

- Клик по строке → переход к детальному просмотру плана (`/admin/curricula/:id`)
- Кнопка перехода к трекам учебного плана

---

### 2. Детали учебного плана (`/admin/curricula/:id`)

**Что отображается:**
- Шапка плана (название, степень, год, ID ИСУ)
- Карточка с информацией (руководитель)
- Кнопка перехода к трекам
- Иерархическое дерево разделов (компонент `SectionItem`)

**API:** `GET /admin/curricula/{id_isu}`

---

### 3. Управление дисциплинами (`/admin/disciplines`)

**Возможности:**
- Просмотр всех дисциплин
- Поиск по названию
- Статистика: всего, найдено

**Для каждой дисциплины:**
- ID, название, комментарий
- Кнопка редактирования (форма обновления)
- Кнопка перехода к графу зависимостей

**CRUD операции:**

| Действие | HTTP | Эндпоинт | Поля |
|----------|------|----------|------|
| Список | `GET` | `/admin/disciplines` | — |
| Обновить | `PUT` | `/admin/disciplines/{disc_id}` | name, comment |

> Создание и удаление дисциплин через UI не реализовано — только редактирование метаданных.

---

### 4. Управление треками (`/admin/tracks/:id`)

**Возможности:**
- Список всех треков учебного плана
- Контекст: название и год учебного плана

**Для каждого трека:**
- Номер и название трека
- ID, лимит выбора
- Кнопки: редактировать, удалить, визуализировать

**CRUD операции:**

| Действие | HTTP | Эндпоинт | Поля |
|----------|------|----------|------|
| Создать | `POST` | `/admin/tracks` | name, number, id_section, count_limit |
| Обновить | `PUT` | `/admin/tracks/{track_id}/edit` | name, number, count_limit |
| Удалить | `DELETE` | `/admin/tracks/{track_id}` | — |

---

### 5. Визуализация трека (`/admin/tracks/:trackId/visualize`)

**Что отображается:**
- Горизонтальная временная шкала по семестрам
- Дисциплины сгруппированы по семестрам (столбцы)
- Цветовая маркировка:
  - Синяя рамка/фон — профильные дисциплины (ФПИ и КТ)
  - Серый фон — общеуниверситетские дисциплины
- Для каждой дисциплины: форма контроля, название, ID

**API:** `GET /admin/tracks/{trackId}/details`

---

### 6. Просмотр данных студентов (как администратор)

Через компонент `StudentSelector` администратор может выбрать любого студента и просмотреть:
- **Рекомендации** (`/`) — рекомендованные дисциплины для выбранного студента
- **Прогресс** (`/progress`) — оценки и статусы дисциплин студента

## Схема базы данных (PostgreSQL, схема s335141)

### Основные таблицы

| Таблица | Ключевые поля | Описание |
|---------|--------------|----------|
| `appuser` | id, login, password, role, student_id | Пользователи системы |
| `students` | id, name, group_id | Студенты |
| `curricula` | id_isu, name, year, degree, head | Учебные планы |
| `sections` | id, id_curricula, id_module, id_parent_section, position | Иерархия разделов |
| `modules` | id_isu, name, type_choose, choose_count | Модули |
| `disciplines` | id, name, comment | Каталог дисциплин |
| `rpd` | id_isu, id_discipline, name, status | Рабочие программы дисциплин |
| `disciplines_in_modules` | id, id_module, id_rpd, position | Дисциплины в модулях |
| `tracks` | id, name, number, count_limit, id_section | Треки специализаций |
| `student_performance` | student_id, discipline_id, grade, status, attempt_number | Успеваемость студентов |
| `discipline_prerequisites` | discipline_id, prerequisite_id | Пререквизиты дисциплин |

### Статусы успеваемости
- `Passed` — сдано
- `Failed` — не сдано (задолженность)
- `Enrolled` — зачислен (в процессе)

## Neo4j граф

### Узлы
`Student`, `Program`, `Track`, `Module`, `Discipline`

### Связи

| Связь | От → До | Описание |
|-------|---------|----------|
| `STUDIES_ON` | Student → Program | Студент обучается по программе |
| `CONTAINS_MODULE` | Program → Module | Программа включает модуль |
| `HAS_MODULE` | Track → Module | Трек включает модуль |
| `PART_OF` | Discipline → Module | Дисциплина входит в модуль |
| `INCLUDES` | Track → Discipline | Трек включает дисциплину (производная связь) |
| `COMPLETED` | Student → Discipline | Студент сдал дисциплину |
| `HAS_DEBT` | Student → Discipline | Академическая задолженность |
| `REQUIRES` | Discipline → Discipline | Пререквизит |

Синхронизация PostgreSQL → Neo4j выполняется скриптом `migrate.py`.
